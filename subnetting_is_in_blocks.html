Subnetting is done always in blocks

If we have an ip 192.168.0.0/23 (255.255.254.0) means 192.168 is the network part and we can choose it to be anything in the range of ip we have been given by the ISP. But for the host part we can choose anything to be a starting ip! As soon the subnet field is not 255, the corresponding ip field starting is alwsys from 0 and follows like this. (/23 subnet means 512 hosts).

  • 192.168.0.0 - 192.168.1.255
    1. hosts: 512
    2. network id (start ip): 192.168.0.0
    3. broadcast ip (end ip): 192.168.1.255
  • 192.168.2.0 - 192.168.3.255
    1. hosts: 512
    2. network id (start ip): 192.168.2.0
    3. broadcast ip (end ip): 192.168.3.255
  • 192.168.4.0 - 192.168.5.255
    1. hosts: 512
    2. network id (start ip): 192.168.4.0
    3. broadcast ip (end ip): 192.168.5.255
  • 192.168.6.0 - 192.168.7.255
    1. hosts: 512
    2. network id (start ip): 192.168.6.0
    3. broadcast ip (end ip): 192.168.7.255
  • etc

Any ip like 192.168.1.23 can't suddenly be told to act as staring ip, it always falls under one these chunks of ip!

To know whether an ip from a particular subnet is the start/end/in between go to: https://www.ipaddressguide.com/cidr
